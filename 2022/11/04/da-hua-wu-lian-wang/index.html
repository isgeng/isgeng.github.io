大话物联网

第一章 物联网又火了

如何判断某种物联网是不是智联网？

  1. 边缘智能

    在终端侧具备基础的边缘智能，在断网离线情况下，可以进行智能决策；

  2. 互联驱动

    智能产品处于“组网”状态时，产品与产品之间能够实现不需要人为干预的智能协同；优先考虑产品间的数据调用，而不是第一时间将数据展示给用户；

  3. 云端升华

    智能产品处于“联网”状态时，云端的人工智能可以完成边缘硬件的数据整合，创造系统与系统之间互相协同的最大价值；

万物互联实现的基础之一在于数据的传输，不同的物联网业务对数据传输能力和实时性都有着不同的要求。

根据传输速率的不同，可将物联网业务进行超高、高、中、低速的区分：

  • 超高速率业务：主要采用4G、5G和Wi-Fi技术，如航拍、直播等，对应的业务特点要求实时数据传输、功耗不敏感、时延短、流量大；
  • 高速率业务：主要采用长期演进(Long Term Evolution, LTE)和高速分组接入(High Speed Packet Access, HSPA)/演进型高速分组接入(HSPA+)技术，如车联网设备、监控摄像头、工业控制和智慧医疗等，对应的业务特点要求功耗不敏感、时延短、流量大；
  • 中速率业务：主要采用增强型机器类通信(Enhanced Machine Type Communication, eMTC)和2G技术，如可穿戴设备、车辆管理、电子广告等，使用频率高但并非实时使用，对应的业务特点要求语音传输、流量功耗较低、覆盖范围广；
  • 低速率业务：主要采用NB-IoT、LoRa和SigFox等低功耗广域网(Low Power Wide Area Network, LPWAN)技术，如无线抄表、智能家居、智能停车等，对应的业务要求传输以文本为主、流量功耗较低、覆盖范围广；

能耗方面：物联网终端工作环境复杂，低功耗是保证物联网持续工作的关键需求之一；

成本方面：物联网终端成本低廉，加入通信模块会使成本显著增加；简单的传感器终端上传的数据量小、信息价值低且使用频次低；低成本是保证物联网大规模部署的关键需求之一；

人们发现任何存储（记账）的需求，都可以套用区块链的机制来进行，这种机制是划时代的。

区块链的六大特征：

  1. 去中心化

    从架构上看，区块链是基于对等网络的；从治理上看，区块链通过共识算法使得少数人很难控制整个系统；架构和治理的去中心化带来三个好处：容错性、抗攻击力和防合谋；

  2. 开放性

    除了交易各方的私有信息被加密外，区块链的数据对所有人公开；

  3. 自治性

    基于公开的算法进行协商，不受任何人为干预；

  4. 去信任

    系统中所有节点之间无需信任也可以进行交易；

  5. 不可篡改性

    信息一旦经过验证并添加至区块链，就会永久地存储起来，除非能够控制系统51%的节点，否则单个节点上对数据库的修改是无效的；

  6. 匿名性

    由于节点之间的交换遵循固定的算法，其数据交互是无需信任的，因而交易对手无需通过公开身份的方式让对方产生信任；

物联网技术发展仍存在一些问题：

  1. 中心化云服务器

    随着连接设备数量的几何级增长，中心化云服务器无法负担；

  2. 不同主体的信任域不同

    各家物联网架构较为封闭，导致不同系统的设备很难实现价值的互联互通；

  3. 传统物联网设备过于陈旧

    固件版本过低、缺少安全补丁、存在权限漏洞等威胁，易受攻击；

应用区块链技术对物联网发展的好处：

  • 降低交易前的验证成本（去信任）
  • 降低运营管理成本（去中心化）
  • 保护数据安全与隐私（不可篡改性）
  • 方便可靠的费用结算和支付

物联网与区块链的结合体——“物链网”，将会给整个社会产业带来更大的想象空间。

第二章 初识物联网

国家标准定义：“通过感知设备，按照约定协议，连接物、人、系统和信息资源，实现对物理和虚拟世界的信息进行处理并作出反应的智能服务系统”。

感知设备是能够获取对象信息并提供接入网络能力的设备，如二维码标签和识读器、电子标签和读写器、摄像头、全球定位系统（Global Positioning System, GPS）、传感器、M2M终端、传感器网关等。

物联网协议是双方实体完成通信或服务所必须遵循的规则和约定，主要包括RFID、红外、ZigBee、蓝牙、3G、4G、5G、Wi-Fi、NB-IoT等通信协议。

物联网中的“物”即物理实体，是能够被物联网感知但不依赖物联网感知而存在的实体；要有相应的数据收发器、要有数据传输信道、要有一定的存储功能、要有中央处理器（Central Processing Unit, CPU）、要有操作系统、要有专门的应用程序、遵循物联网的通信协议、在世界网络中有可被识别的唯一标识。

智能服务是能够自动识别用户的显性和隐形需求，并主动、高效、安全、绿色地满足其需求的服务。它实现的是一种按需和主动的智能，即通过捕捉用户的原始信息，通过后台积累的数据，构建需求结构模型，进行数据挖掘和商业智能分析，除了可以分析用户的习惯、喜好等显性需求外，还可以进一步挖掘与时空、身份、工作生活状态关联的隐性需求，主动给用户提供精准、高效的服务。这里需要的不仅仅只是传递和反馈数据，更需要系统进行多维度、多层次的感知和主动、深入的识别。

与互联网相比，物联网实际上只是多了一个底层的数据采集环节，大致是4类数据的采集：电子标签显示身份、传感器捕捉姿态、摄像头记录图像、GPS进行追踪定位。

一般来说，物联网的工作原理是：

  • 首先，对物体属性进行标识，属性包括静态和动态的属性，静态属性可以直接存储在标签中，动态属性需要先由传感器实时进行探测；
  • 其次，需要识别设备完成对物体属性的读取，并将信息转换为蛇和网络传输的数据格式；
  • 最后将物体的信息通过网络传输到信息处理中心，由信息处理中心完成物体通信的相关计算，处理中心可能是分布式的，也可能是集中式的，如电信运营商的互联网数据中心（IDC, Internet Data Center）；

第三章 不以规矩，不能成方圆

物联网涉及的标准化组织十分复杂，既有国际、区域和国家标准化组织，又有行业协会和联盟组织。依据物联网的参考体系结构和技术框架，不同标准组织侧重的技术领域也不同，有些标准组织的工作覆盖多个层次，不同标准组织之间错综交互。

从射频识别（RFID）、机器对机器（M2M）、机器类通信（MTC，Machine Type Communication）、无线传感网（WSN，Wireless Sensor Networks），到泛在网（UN，Ubiquitous Networks）、物联网（IoT）、泛在传感网（USN，Ubiquitous Sensor Network），国内外标准组织开展了与物联网相关的大量标准工作。

第四章 感知层：物联网的皮肤和五官

根据技术框架，物联网可分为三层：

  1. 感知层：相当于人体的皮肤和五官；
  2. 网络层：相当于人体的神经中枢和大脑；
  3. 应用层：相当于人的社会分工；

感知层包括条码和扫描器、RFID标签和读写器、摄像头、GPS、传感器、传感器网络等。其中条码和RFID标签显示身份，传感器捕捉状态，摄像头记录图像，GPS进行跟踪定位，最终实现识别物体、采集信息的目标。

4.1 条码：物联网的第一代身份证

条码——商品外包装上一组黑白相间的条纹。

4.2 电子标签：物联网的第二代身份证

随着射频识别（RFID）的发展和普及，贴有电子标签的商品随处可见。窄带物联网（NB-IoT）和5G（第五代移动通信）进一步推动了基于RFID的物联网发展。

相较于传统条码识别技术，射频识别的优势主要表现在：1.能够为每件物品分配唯一的标识；2.批量识别，扫描速度快；3.体积小，易封装；4.抗污染能力强，可实现穿透性识别；5.可重复使用；6.穿透性和无屏障阅读；7.数据的存储容量大；8.安全性；

4.3 传感器：物联网的神经元

物联网需要对物体有全面感知的能力，对信息具有互联互通的能力，并对系统具有智慧计算的能力，从而形成一个连接人与物体的信息网络。

传感器技术、通信技术和计算机技术并称为信息技术的三大支柱，它们构成了信息系统的“感官”“神经”和“大脑”，分别用于完成信息的采集、传输和处理。

智能手机的传感器种类包括：重力/加速度传感器、方向传感器、陀螺仪、GPS、距离传感器、光线传感器、图像传感器、指纹识别传感器、气压传感器、湿度传感器、温度传感器、声音传感器、磁场传感器、紫外线传感器、心率传感器、血氧传感器等。

第五章 网络层：物联网的神经中枢和大脑

网络层包括通信与互联网的融合网络、网络管理中心、信息中心和智能处理中心等，涉及的关键技术包括光通信网络、移动通信网、异构网、虚拟专用网（VPN，Virtual Private Network）、互联网、M2M网络、局域网、Wi-Fi、自组织网络和总线网等。

  1. 互联网：阿帕网——万维网——Web 2.0

  2. 移动通信网：第六代移动通信系统（6G，6th Generation）概念研究的启动和5G商用步伐的加快，以及移动通信业务范围的不断扩大，为物联网深入应用奠定了坚实的技术基础。由于物联网节点具有分散性、移动性和海量性等特点，这就决定了移动通信是实现物联网可靠传输的主要技术手段。

  3. ZigBee：与蓝牙类似的一种短距离无线通信技术；除了Wi-Fi、蓝牙之外，ZigBee是目前最重要的无线通信协议之一，主要应用于物联网和智能硬件等领域。

    当前众多智能设备都采用了Wi-Fi和蓝牙技术，小米为什么看上了非主流的ZigBee协议呢？因为ZigBee是唯一完整、可互操作的物联网解决方案，涵盖网状网络层到通用语言层，使智能产品能够协同工作。

    ZigBee支持两种具有不同通信能力的设备：全功能设备（FFD，Full Function Device）和简化功能设备（RFD，Reduce Function Device）。FFD之间及FFD与RFD之间都可通信；RFD之间不可通信，只能与FFD通信，或通过FFD向外转发数据。

    ZigBee网络可组成星型网络或对等网络。

    • 在星型网络中，所有普通设备都与中心设备（协调器）直接通信。
    • 在对等网络中，只要彼此都在对方的无线辐射范围之内，任何两台设备之间都可以直接通信。

第六章 应用层：物联网的“加工厂”

应用层是物联网与行业专业技术的深度融合，与行业需求结合，实现行业智能化。

应用层涉及的关键技术可分为应用设计、应用支撑、终端设计三个子类。其中应用支撑子类中包括云计算、大数据和人工智能技术。
